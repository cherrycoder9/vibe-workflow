# GET vs POST Form Submission

이미 봤겠지만, 폼 액션을 호출하려면 `method="POST"`를 써야 해.

근데 어떤 폼들은 서버에 데이터를 POST 할 필요가 없어. 예를 들면 검색창 같은 거 말이지. 이런 경우에는 `method="GET"`을 쓰거나, 아예 `method` 속성을 안 써도 돼 (결국 똑같음). SvelteKit은 이런 폼들을 마치 `<a>` 태그처럼 취급해서, 페이지 전체를 새로고침하는 대신 클라이언트 사이드 라우터를 사용해:

```svelte
<form action="/search">
	<label>
		검색
		<input name="q">
	</label>
</form>
```

이 폼을 제출하면 `/search?q=...` 같은 주소로 이동하고, 해당 페이지의 `load` 함수를 호출하지만, 액션(action)은 호출하지 않아. `<a>` 태그와 마찬가지로, `<form>` 태그에 `data-sveltekit-reload`, `data-sveltekit-replacestate`, `data-sveltekit-keepfocus`, `data-sveltekit-noscroll` 같은 속성을 설정해서 라우터의 동작을 제어할 수 있어. "GET 폼은 그냥 링크 누르는 거랑 비슷하다고 보면 된다, 이 말이야."

---

얘 뭐 하는 애냐?
HTML 폼에서 데이터를 서버로 보낼 때 쓰는 두 가지 주요 방식, GET과 POST의 차이점을 설명하고, 특히 SvelteKit 프레임워크에서 각각 어떻게 다르게 처리되는지 알려주는 내용이야. "GET이냐 POST냐, 그것이 문제로다."

왜 쓰는데? (왜 이런 구분이 필요한데?)
1.  **GET의 목적**: 주로 서버에서 데이터를 가져오거나(Read), 검색할 때 써. URL에 쿼리 파라미터 형태로 데이터가 붙어서 전송되기 때문에, 사용자가 그 URL을 북마크하거나 다른 사람에게 공유하기 편해. "이 검색 결과 너도 봐봐!" 할 때 좋지. 서버 상태를 변경시키지 않는 요청에 적합해 (멱등성).
2.  **POST의 목적**: 서버에 새로운 데이터를 생성(Create)하거나, 기존 데이터를 수정(Update) 또는 삭제(Delete)할 때 주로 사용돼. 데이터가 HTTP 요청 본문(body)에 담겨 전송되므로 URL에 노출되지 않고, 대용량 데이터나 파일 전송도 가능해. "회원가입 정보나 게시글 내용 같은 건 POST로 보내야지, 주소창에 다 뜨면 큰일 나."
3.  **SvelteKit의 처리 방식**: SvelteKit은 이런 표준적인 용도에 맞춰 GET 폼은 페이지 이동 및 데이터 로딩(`load` 함수)과 유사하게, POST 폼은 서버 측 로직 실행(액션)과 연결되도록 설계했어.

언제 불려 나오냐? (언제 어떤 방식을 쓰냐?)
*   **GET**:
    *   검색창처럼 사용자가 입력한 값으로 특정 정보를 조회하고 싶을 때.
    *   필터링 옵션을 URL로 표현하고 싶을 때.
    *   서버의 데이터를 변경하지 않고, 단순히 특정 페이지나 상태로 이동하고 싶을 때.
    *   SvelteKit에서는 해당 라우트의 `load` 함수를 실행시켜 페이지에 필요한 데이터를 불러오는 데 사용돼.
*   **POST**:
    *   회원가입, 로그인, 글쓰기, 댓글 달기 등 서버의 데이터를 생성하거나 변경해야 할 때.
    *   사용자 아이디, 비밀번호 같은 민감한 정보를 전송할 때 (URL에 노출되면 안 되니까).
    *   파일 업로드처럼 전송할 데이터의 크기가 클 때.
    *   SvelteKit에서는 `+page.server.js` 파일에 정의된 `actions` 객체 내의 특정 함수를 실행시키는 데 사용돼.

쓸 때 꿀팁 및 주의사항:
*   **GET은 URL 노출**: GET 요청은 데이터가 URL에 그대로 드러나. 그러니까 비밀번호 같은 민감 정보는 절대 GET으로 보내면 안 돼. "주소창에 `?password=1234` 박제하고 싶지 않으면 명심해라."
*   **POST는 액션 전용**: SvelteKit에서 `method="POST"`인 폼은 해당 라우트의 `+page.server.js` 파일에 정의된 서버 액션(action)을 실행하기 위한 거야.
*   **GET은 `load` 함수와 연동**: `method="GET"` (또는 생략) 폼을 제출하면 URL이 바뀌고, SvelteKit 라우터는 이 변경된 URL을 기반으로 해당 페이지의 `load` 함수를 호출해서 데이터를 가져오거나 페이지를 렌더링해. 액션은 실행 안 돼.
*   **`data-sveltekit-*` 속성 활용**: GET 폼도 `<a>` 태그처럼 라우팅 동작을 세밀하게 제어할 수 있어. 예를 들어 `data-sveltekit-noscroll`을 쓰면 페이지 이동 후 스크롤 위치를 그대로 유지할 수 있지. "디테일이 중요하다."
*   **기본값은 GET**: `<form>` 태그에 `method` 속성을 따로 지정하지 않으면 기본적으로 GET 방식으로 동작해. "깜빡하면 GET으로 간다."
*   **멱등성**: GET 요청은 여러 번 보내도 서버 상태가 변하지 않고 같은 결과를 반환해야 해(멱등성). POST는 호출할 때마다 서버 상태가 바뀔 수 있어. "검색은 백 번 해도 똑같지만, 글쓰기는 백 번 하면 글 백 개 생기는 거랑 같은 이치."

`text()`
텍스트를 `Content-Type:text/plain`으로 렌더링합니다.

```javascript
app.get('/say', (c) => {
  return c.text('Hello!')
})
```

---

**얘 뭐 하는 애냐?**
`c.text()`는 Hono 핸들러 안에서 클라이언트에게 순수한 텍스트 문자열을 응답으로 보내고 싶을 때 쓰는 아주 간단명료한 녀석입니다. 호출하면 알아서 `Content-Type` 헤더를 `text/plain;charset=UTF-8`으로 설정해주고, 전달한 문자열을 응답 본문으로 삼아 퉤 뱉어줍니다. "야, 그냥 이 글자 그대로 보내줘!" 할 때 딱이죠.

**왜 쓰는데?**
1.  **세상 간단한 응답**: "OK", "PONG" (헬스 체크 응답 같은 거), 간단한 메시지, 디버깅용 텍스트 등 복잡한 구조 없이 그냥 글자 몇 개 툭 던져주고 싶을 때 최고입니다. HTML 태그니 JSON 형식이니 신경 쓸 필요가 없어요.
2.  **`Content-Type` 자동완성**: `c.header('Content-Type', 'text/plain')` 이런 코드 안 쳐도 알아서 해주니 편합니다. 개발자는 그냥 보낼 텍스트만 신경 쓰면 되죠. "귀차니즘은 개발자의 미덕!"
3.  **HTML/JSON 아닌 진짜 텍스트**: 가끔은 정말 꾸밈없는 순수 텍스트가 필요할 때가 있습니다. 예를 들어, 다른 스크립트가 파싱하기 편한 형태로 데이터를 내려주거나, 아주 단순한 API 엔드포인트를 만들 때 유용합니다.

**언제 불려 나오냐?**
Hono 라우트 핸들러 함수의 맨 마지막, 즉 "이제 진짜 응답 보낸다!" 하는 시점에 `return c.text('보낼 메시지')` 형태로 호출됩니다. 얘가 호출되면 Hono는 응답 처리를 마무리하고 클라이언트에게 결과를 전송합니다.

**쓸 때 꿀팁 및 주의사항:**
*   **문자열 전용**: 이름 그대로 텍스트만 받습니다. 객체나 배열 같은 거 넣으면 `"[object Object]"` 같은 슬픈 문자열로 변해서 나갑니다. "글자만 달라, 글자만!"
*   **상태 코드 변경 가능**: `c.text('Not Found', 404)`처럼 두 번째 인자로 HTTP 상태 코드를 넘겨줄 수 있습니다. 기본값은 당연히 `200 OK`고요.
*   **`Content-Type` 커스텀? 굳이?**: `c.text()`는 `text/plain`을 강제하는 편의 기능입니다. 만약 `text/csv`나 `text/html` 같은 다른 `text/*` 타입을 보내고 싶다면, `c.header('Content-Type', 'text/csv')`로 설정하고 `c.body('CSV,내용')`을 쓰는 게 더 명확합니다. `c.text()` 쓰면서 굳이 `Content-Type`을 바꾸려 하면 "얘 왜 이러나?" 싶을 수 있어요.
*   **보안? `text/plain`은 비교적 안전**: `text/plain`으로 응답하면 브라우저는 이걸 HTML로 해석하지 않아서, 사용자 입력값을 그대로 출력해도 XSS(Cross-Site Scripting) 공격에 비교적 안전합니다. 그래도 사용자 입력을 아무 생각 없이 뱉는 건 좋은 습관이 아니니, "혹시 모르니 조심하자" 정도는 생각하는 게 좋습니다.
*   **인코딩은 UTF-8이 국룰**: 기본적으로 `charset=UTF-8`로 나가는데, 이걸 바꾸고 싶다면 `c.header()`로 `Content-Type` 헤더를 직접 조작해야 합니다. 근데 21세기에 UTF-8 말고 다른 거 쓸 일이 얼마나 있겠어요?
*   **`c.body()`와의 관계**: `c.body(string)`도 문자열을 보낼 수 있지만, `c.body()`는 좀 더 로우레벨입니다. `c.text()`는 `c.body()`를 사용하면서 `Content-Type`까지 알아서 챙겨주는 친절한 형님 같은 존재죠.
